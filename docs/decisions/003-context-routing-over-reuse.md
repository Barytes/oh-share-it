# D-003：把“复用”改写成 context routing 问题

**状态**：已采纳

**日期**：2026-05-01

## 背景

“复用”是有价值的结果，但如果产品过度围绕复用设计，会暗示所有 context 都应该被公共化、标准化、抽象化，并被尽可能多的人使用。

这不符合真实团队知识的形态。很多 context 只在特定人、特定项目、特定时间、特定问题下有用。

## 决策

产品核心问题应从“如何让 context 被更多人复用”改为：

**当一个人或 agent 面对某个问题时，系统如何路由到最相关、最可信、最合适边界的 context。**

## 产品含义

context 的价值不由“被多少人复用”单独决定，而由它是否在正确场景中被正确调用决定。

好的路由需要考虑：

- 当前问题是什么。
- 提问者是谁。
- 哪些人或项目拥有相关 context。
- context 的权限和共享边界是什么。
- context 是否过期。
- context 的来源和置信度是什么。
- 是否存在冲突观点或多个有效视角。
- agent 是否需要原文、摘要、证据链还是行动建议。

## 设计约束

- 北极星指标不应只是内容量或浏览量，而应关注成功路由后的有效帮助。
- 产品体验应鼓励用户保留 context 边界，而不是逼迫他们把所有东西整理成公共文档。
- 搜索、问答和推荐都应被视为 routing surface。
