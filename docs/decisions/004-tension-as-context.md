# D-004：冲突观点是高价值 context，不应被默认合并

**状态**：已采纳

**日期**：2026-05-01

## 背景

复杂知识工作中，不同成员可能基于不同经验形成不同判断。普通知识库和 RAG 系统容易把这些差异压缩成一个看似统一的答案。

但很多时候，冲突本身就是团队最需要保留的知识。

## 决策

`oh-share-it` 应把冲突、分歧和 tension 视为一等 context。

当系统发现不同 context 之间存在不一致时，不应默认进行 majority-wins 式合并，而应保留不同来源、观点、证据和适用边界。

## 产品含义

- 用户可以看到某个问题下有哪些不同判断。
- agent 回答时应能说明“这里存在不同观点”。
- 冲突可以被整理成 tension，但 tension 不必强制解决。
