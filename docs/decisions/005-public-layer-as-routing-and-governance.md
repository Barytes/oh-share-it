# D-005：公共层更像路由层和治理层，而不是单一真相层

**状态**：已采纳

**日期**：2026-05-01

## 背景

如果把公共知识库理解成“团队唯一真相来源”，产品会天然走向中心化、合并、审核和标准答案。

但 `oh-share-it` 想处理的是多人 context 的边界、来源和场景调用，而不是制造一个全知的统一文档库。

## 决策

公共层的核心职责是：

- 发现哪些 context 可能相关。
- 管理 context 的来源、权限和边界。
- 帮助人和 agent 在不同 context 空间之间路由。
- 保留公共可见的索引、贡献、引用、tension 和审计记录。

公共层不是所有个人 context 的终点，也不是唯一真相层。

## 产品含义

- 公共知识不一定复制个人知识全文，可以保存索引、摘要、引用关系和访问路径。
- 有些 context 只可被请求、引用或临时授权，不必永久进入公共池。
- “团队资产”不是一个中央大文档，而是一套可路由、可追溯、可治理的 context 网络。
